Domain String / Text

Note: Value type DomainText is not described here as it works the same as DomainString, except for the limitless text length.

String Value Types extend DomainString, which is a wrapper arround a String instance. The constructor requires a meta-object MString to specify length and case.

The string value-type-class must implement at least one method; createInstance(). This method allows the return of typed objects after operations as ‘append’.

For example, the declaration of a new Description value type with a lenght of 60 characters.

class Description extends DomainString<Description>{
	
	public Description(final String value) {
		super(MString.create(60, false), value);
	}

	@Override
	protected Description createInstance(final String value) {
		return new Description(value);
	}
}

usage:

final Description d1 = new Description("abcdefg");

System.out.println("d1.append('_hij'): " +d1.append("_hij"));
System.out.println("getLeft(3)       : " +d1.getLeft(3));
System.out.println("isEmpty()        : " +d1.isEmpty());
System.out.println("isNotEmpty()     : " +d1.isNotEmpty());

result:

d1.append('_hij'): abcdefg_hij
getLeft(3)       : abc
isEmpty()        : false
isNotEmpty()     : true

Note that strings are not automatically truncated, if they do not fit the domain, an exception will be thrown:

new Description("abcdefg".repeat(30));

result:

DomainValueException: Value exceeds 60 characters.

Equality

// Equality is based on the underlying Boolean value.
final Name n1 = new Name("abcdef");
final Description d1 = new Description("abcdef");

System.out.println("n1.equal(d1) : " + n1.equals(d1));
System.out.println("d1.equal(d2) : " + d1.equals(d2));

result:

n1.equal(d1) : true
d1.equal(d2) : true
