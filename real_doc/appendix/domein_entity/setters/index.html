Setters

Entities have two setter methods for every attribute field:

  • set<attributeName>(<ValueType>): sets a DomainValue
  • set<attributeName>Value(<primitive-value>): sets the underlying Java value

one setter method for relation fields:

  • set<relationName>(<DomainEntity> ): sets a DomainEntity

and one generic setFieldValue("<fieldName>") method in order to set field values by name.

The value-parameter types for setters are as follows:

ValueType @Override set<fieldName>(...) final set<fieldName>Value(..) Nullable
DomainBoolean ValueType Boolean No
DomainDate ValueType LocalDate Yes
DomainDateTime ValueType LocalDateTime Yes
DomainDecimal ValueType BigDecimal No
DomainEnumerate Enum String (code) No
DomainInteger ValueType Integer No
DomainLong ValueType Long No
DomainString ValueType String No
DomainText ValueType String Yes
DomainTime ValueType LocalTime Yes
DomainRelation DomainEntity undefined Yes

For relation-fields the method setFieldValue(final String fieldName, Object value) expects a DomainEntity value.

Read-only fields do have setters for internal use, however these setters are prefixed with underscores “__ “ to indicate that they are not part of the public API of the DomainEntity.
