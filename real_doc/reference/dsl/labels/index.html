Labels

Labels enable the internationalization (i18n) of the user-interface. All user-facing text is replaced by label-codes. These codes are grouped together into label-sets, and translated using the standard Java i18n property-files.

For example, DSL declaration:

package nl.openframe.erp.system.bt

labelSet SYS_BTLabels{

	label timeStampLabel "Time Stamp"
	label emailLabel "Email" icon "EMAIL"
	label confirmedLabel "Confirmed"
}

The label set has an identifier (SYS_BT_Labels), and every label has a code (confirmedLabel, etc.). Together with the package name, these comprise the fully qualified name of the label. The confirmedLabel, for example, is imported in the following Confirmed value type declaration:

package nl.openframe.erp.system.bt

import nl.openframe.erp.system.bt.SYS_BTLabels.confirmedLabel

boolean Confirmed confirmedLabel

Labels can optionally have a short name, an icon (Material Design Icons) and a style code. The style code is generally used for colors, for example:

label alertStatusLabel1 "Bug" icon "BUG_REPORT" style "red"
label alertStatusLabel2 "Error"	icon "BUG_REPORT" style "red"
label alertStatusLabel3 "Warning" icon "BUG_REPORT" style "orange"
label alertStatusLabel4 "Info" icon "BUG_REPORT" style "blue"
label alertStatusLabel5 "Solved" icon "BUG_REPORT" style "green"

Generated code

Label-sets generate two source files:

  1. A .properties file with one or more entries per label, which can be translated according to standard Java practises.
timeStampLabel=Time Stamp
emailLabel=Email
emailLabel.icon=EMAIL
confirmedLabel=Confirmed
  1. A java class with a constant for every label name. This alows you to reference labels without the use of string literals.
package nl.openframe.erp.system.bt;

public class SYS_BTLabels{

	public static final String TIME_STAMP_LABEL = "timeStampLabel";
	public static final String EMAIL_LABEL = "emailLabel";
	public static final String CONFIRMED_LABEL = "confirmedLabel";
}

Labels also have optional properties for ‘focus-icon’ and ‘focus-style’, be these are currently not used.

All labels are imported into the application, and for convenience reasons, they are referenced only by label name. This means that label names should be globally unique in order to avoid conflicts.

The generated files are located in the ‘generated’ directory. They will be overwritten during every code-generation and should not be changed manually. The label-set should hold the default language text, for extra languages new ‘.properties’ files should be added in the ‘manual’ code directory.
